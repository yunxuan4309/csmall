# CSmall 微服务模块启动顺序

## 前置基础设施

以下服务需在启动任何业务模块之前先运行：

| 顺序 | 服务 | 端口 | 说明 |
|------|------|------|------|
| 0.1 | **Nacos** | 8848 | 注册/配置中心，所有业务服务都依赖 |
| 0.2 | **MySQL** | 3306 | 数据库：cs_mall_ams, cs_mall_ums, cs_mall_pms, cs_mall_oms, cs_mall_seckill, leafdb |
| 0.3 | **Redis** | 6379 | 缓存，大多数服务依赖 |
| 0.4 | **Seata** | 8091 | 分布式事务，商品和订单服务需要 |
| 0.5 | **RabbitMQ** | 5672 | 消息队列，秒杀服务需要 |

| 0.6 | **Elasticsearch** | 9200 | 搜索引擎，搜索服务需要 |
| 0.7 | **Sentinel** | 8080 | 流量控制，秒杀服务需要（可延后） |

---

## 第一阶段：独立服务（无 Dubbo 依赖，可并行启动）

| 顺序 | 服务 | 端口 | 依赖 | 说明 |
|------|------|------|------|------|
| 1.1 | **leaf-server** | 9090 | MySQL | 分布式ID生成器，仅需 MySQL 已经弃用
| 1.2 | **mall-resource** | 9060 | 无 | 文件上传服务，完全独立 |
| 1.3 | **mall-ams-webapi** | 10003 | Nacos + MySQL + Redis | 后台管理服务，无 Dubbo 消费 |
| 1.4 | **mall-ums-webapi** | 10006 | Nacos + MySQL + Redis | 用户管理服务，Dubbo 提供者但暂无消费方 |

---

## 第二阶段：核心 Dubbo 提供者（必须在第三、四阶段之前启动）

| 顺序 | 服务 | 端口 | 依赖 | 说明 |
|------|------|------|------|------|
| 2.1 | **mall-product-webapi** | 9010 | Nacos + MySQL + Redis + Seata | **核心服务**，提供 7 个 Dubbo 接口，被多个服务调用 |

**mall-product-webapi 提供的 Dubbo 接口：**

- `IForFrontSpuService` — 前台商品SPU查询（被 mall-front、mall-search、**mall-ai** 调用）
- `IForFrontSkuService` — 前台商品SKU查询
- `IForFrontCategoryService` — 前台分类查询
- `IForFrontAttributeService` — 前台属性查询
- `IForOrderSkuService` — 订单扣减库存
- `IForSeckillSpuService` — 秒杀SPU查询
- `IForSeckillSkuService` — 秒杀SKU查询
- `ISpuSyncService` — 商品变更同步到 ES（mall-product → mall-ai 触发）

---

## 第三阶段：依赖商品服务的消费者（第二阶段之后，可并行启动）

| 顺序 | 服务 | 端口 | 依赖 | 调用的 Dubbo 服务 |
|------|------|------|------|------|
| 3.1 | **mall-order-webapi** | 10005 | Nacos + MySQL + Redis + Seata | → mall-product (IForOrderSkuService) |
| 3.2 | **mall-front-webapi** | 10004 | Nacos + Redis | → mall-product (IForFrontSpuService, IForFrontSkuService, IForFrontCategoryService, IForFrontAttributeService) |
| 3.3 | **mall-search-webapi** | 10008 | Nacos + Redis + Elasticsearch | → mall-product (IForFrontSpuService) |
| 3.4 | **mall-ai-webapi** | 10010 | Nacos + Redis + Elasticsearch | → mall-product (IForFrontSpuService, ISpuSyncService) |

---

## 第四阶段：依赖商品+订单服务的消费者（第三阶段之后启动）

| 顺序 | 服务 | 端口 | 依赖 | 调用的 Dubbo 服务 |
|------|------|------|------|------|
| 4.1 | **mall-seckill-webapi** | 10007 | Nacos + MySQL + Redis + RabbitMQ + Sentinel | → mall-product (IForSeckillSpuService, IForSeckillSkuService) + mall-order (IOmsOrderService) |

---

## 第五阶段：网关与认证（业务服务就绪后启动）

| 顺序 | 服务 | 端口 | 依赖 | 说明 |
|------|------|------|------|------|
| 5.1 | **mall-sso** | 10009 | Nacos + MySQL(ams+ums) + Redis | 单点登录，直连 ams/ums 数据库 |
| 5.2 | **mall-gateway-server** | 10087 | Nacos | API 网关，通过 Nacos 路由到各服务 |

---

## 不可启动的模块（纯 JAR 依赖，无启动类）

以下模块是库模块，仅作为 Maven 依赖被引用，不需要单独启动：

- mall-common — 公共工具库
- mall-pojo — 数据模型库
- mall-ams-service — AMS 接口+实现
- mall-ums-service — UMS 接口+实现
- mall-product-service — 商品接口+实现
- mall-order-service — 订单接口+实现
- mall-seckill-service — 秒杀接口+实现
- mall-search-service — 搜索接口
- mall-front-service — 前台接口
- mall-ai-service — AI 导购接口
- leaf-core — Leaf 核心库
