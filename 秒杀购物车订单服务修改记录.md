## 第二轮修改（2026-04-15）

### 7. 购物车删除失败（NOT_FOUND）

**问题**：用户添加购物车正常，但删除和清空购物车均失败，抛出 `CoolSharkServiceException(responseCode=NOT_FOUND)`。

排查发现三个层面的问题：

#### 7.1 实体类缺少 `@TableId` 注解

order、ums、seckill、product 模块的实体类 `id` 字段均缺少 `@TableId` 注解，导致 MyBatis-Plus 在执行 `delete`、`updateById` 等操作时无法正确识别主键字段。

**涉及文件（共22个）**：

- order 模块：`OmsCart.java`、`OmsOrder.java`、`OmsOrderItem.java`
- ums 模块：`User.java`、`UserDetail.java`、`UserLoginLog.java`、`DeliveryAddress.java`、`RewardPointLog.java`、`ChangePasswordLog.java`
- seckill 模块：`Success.java`、`SeckillSpu.java`、`SeckillSku.java`
- product 模块：`Spu.java`、`SpuDetail.java`、`Sku.java`、`SkuSpecification.java`、`Category.java`、`Brand.java`、`Picture.java`、`Album.java`、`Attribute.java`、`AttributeTemplate.java`、`AttributeTemplateValue.java`、`BrandCategory.java`、`CategoryAttributeTemplate.java`

**修复**：为所有实体类添加 `@TableId(value = "id", type = IdType.ASSIGN_ID)` 注解。

#### 7.2 删除方法未验证用户归属

`OmsCartServiceImpl.removeCart()` 使用 `deleteBatchIds` 直接按 ID 删除，未验证购物车商品是否属于当前登录用户，存在越权删除安全隐患。

**修复**：改用 `LambdaQueryWrapper` 添加 `userId` 条件：

```java
LambdaQueryWrapper<OmsCart> wrapper = new LambdaQueryWrapper<>();
wrapper.in(OmsCart::getId, Arrays.asList(ids))
       .eq(OmsCart::getUserId, userId);
int rows = omsCartMapper.delete(wrapper);
```

#### 7.3 JavaScript Long 类型精度丢失（核心问题）

雪花算法生成的 ID 超出 JavaScript `Number` 类型的安全整数范围，前端接收后末尾数字丢失，传回后端时无法匹配数据库记录。

**修复**：在 `mall-common` 模块新建 `JacksonConfiguration.java`，全局将 `Long` 类型序列化为字符串：

```java
@Configuration
public class JacksonConfiguration {
    @Bean
    public Jackson2ObjectMapperBuilderCustomizer jackson2ObjectMapperBuilderCustomizer() {
        return builder -> {
            builder.serializerByType(Long.class, ToStringSerializer.instance);
            builder.serializerByType(Long.TYPE, ToStringSerializer.instance);
        };
    }
}
```

同时在 `MallCommonConfiguration.java` 的 `@ComponentScan` 中添加 `com.cooxiao.mall.common.config` 包路径。

## 经验总结

1. **实体类主键注解**：使用 MyBatis-Plus 时，实体类必须添加 `@TableId` 注解标注主键
2. **接口安全**：删除操作应验证数据归属，防止越权操作
3. **Long 精度问题**：雪花算法生成的 ID 传给前端时，必须序列化为字符串，避免 JavaScript 数值精度丢失
