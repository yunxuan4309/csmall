# Spring Boot 2.x → 3.x 技术栈迁移问题总结（第二轮）

## 问题背景

在完成第一轮迁移后，项目进行了深入开发。在后续的代码审查中，又发现了一批 Spring Boot 3.x 升级带来的兼容性问题，现已全部解决。

## 一、Security 配置类重复定义问题

### 1.1 问题描述

```
Duplicate class: 'ResourceWebSecurityConfiguration'
Cannot resolve method 'cors()' in 'HttpSecurity'
```

### 1.2 根本原因

部分模块的 `ResourceWebSecurityConfiguration.java` 文件包含了两个版本的代码：
- 新版 Spring Security 6.x 的 `SecurityFilterChain` Bean
- 旧版 Spring Security 5.x 的 `configure(HttpSecurity)` 方法

这两种写法不能共存于同一个类中。

### 1.3 解决方案

删除旧版残留代码，统一使用 Spring Security 6.x 的配置方式：

```java
// 错误示例：类中包含两套配置
public class ResourceWebSecurityConfiguration {
    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) {
        // ... 新代码
    }
}
// 以下是残留的旧代码，会导致编译错误
@Override
protected void configure(HttpSecurity http) throws Exception {
    // ... 旧代码
}
```

```java
// 正确写法：只保留一套
@Configuration
@EnableWebSecurity
@EnableMethodSecurity(prePostEnabled = true)
public class ResourceWebSecurityConfiguration {

    @Autowired
    private SSOFilter ssoFilter;
    @Autowired
    private MyAccessDeniedHandler myAccessDeniedHandler;
    @Autowired
    private MyAuthenticationEntryPoint myAuthenticationEntryPoint;

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }

    @Bean
    CorsConfigurationSource corsConfigurationSource() {
        CorsConfiguration configuration = new CorsConfiguration();
        configuration.setAllowedOrigins(Arrays.asList("*"));
        configuration.setAllowedHeaders(Arrays.asList("*"));
        configuration.setAllowedMethods(Arrays.asList("*"));
        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/**", configuration);
        return source;
    }

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http.csrf(csrf -> csrf.disable());
        http.cors(cors -> cors.configurationSource(corsConfigurationSource()));
        http.authorizeHttpRequests(auth -> auth
                .requestMatchers("/", "/*.html", "/favicon.ico",
                        "/**/*.html", "/**/*.css", "/**/*.js",
                        "/swagger-resources/**", "/v2/api-docs/**",
                        "/v3/api-docs/**", "/doc.html").permitAll()
                .anyRequest().authenticated()
        );
        http.exceptionHandling(exception -> exception
                .accessDeniedHandler(myAccessDeniedHandler)
                .authenticationEntryPoint(myAuthenticationEntryPoint)
        );
        http.sessionManagement(session -> session.sessionCreationPolicy(SessionCreationPolicy.STATELESS));
        http.addFilterBefore(ssoFilter, UsernamePasswordAuthenticationFilter.class);
        return http.build();
    }
}
```

### 1.4 CORS API 变更

**问题描述**：
```
'cors()' is deprecated since version 6.1 and marked for removal
```

**根本原因**：
Spring Security 6.1 开始弃用了部分 CORS API，推荐使用新方式。

**解决方案**：
```java
// 修复前（已弃用）
corsConfiguration.addAllowedOrigin("*");
corsConfiguration.addAllowedHeader("Authorization");
corsConfiguration.addAllowedMethod("*");

// 修复后
configuration.setAllowedOrigins(Arrays.asList("*"));
configuration.setAllowedHeaders(Arrays.asList("*"));
configuration.setAllowedMethods(Arrays.asList("*"));
```

## 二、MyBatis 拦截器配置类 jakarta 迁移

### 2.1 问题描述

```
Cannot resolve symbol 'PostConstruct'
```

### 2.2 根本原因

Spring Boot 3.x 基于 Jakarta EE 9+，所有 `javax.*` 包迁移为 `jakarta.*`。

### 2.3 解决方案

```java
// 修复前
import javax.annotation.PostConstruct;

// 修复后
import jakarta.annotation.PostConstruct;
```

## 三、Apache Commons Lang 包名变更

### 3.1 问题描述

```
Cannot resolve symbol 'StringUtils'
```

### 3.2 根本原因

Apache Commons Lang 从 2.x 升级到 3.x 后，包名从 `org.apache.commons.lang` 变为 `org.apache.commons.lang3`。

### 3.3 解决方案

```java
// 修复前
import org.apache.commons.lang.StringUtils;

// 修复后
import org.apache.commons.lang3.StringUtils;
```

## 四、Nacos RandomUtils 替换

### 4.1 问题描述

```
Cannot resolve symbol 'RandomUtils'
```

### 4.2 根本原因

`com.alibaba.nacos.client.naming.utils.RandomUtils` 在新版 Nacos 客户端中已移除或路径变更。

### 4.3 解决方案

使用 Java 标准库的 `ThreadLocalRandom` 替代：

```java
// 修复前
import com.alibaba.nacos.client.naming.utils.RandomUtils;
RandomUtils.nextInt(10000)

// 修复后
import java.util.concurrent.ThreadLocalRandom;
ThreadLocalRandom.current().nextInt(10000)
```

### 4.4 优势说明

| 方面 | RandomUtils | ThreadLocalRandom |
|------|-------------|-------------------|
| 依赖 | Nacos 客户端 | Java 标准库 |
| 线程安全 | 有锁竞争 | 无锁（线程本地） |
| 性能 | 一般 | 更优 |
| 随机范围 | nextInt(n) | nextInt(n) |

**逻辑完全一致**，返回值范围相同。

## 五、MyBatis Plus Generator API 重构

### 5.1 问题描述

```
'AutoGenerator()' has private access in 'com.baomidou.mybatisplus.generator.AutoGenerator'
Cannot resolve method 'setGlobalConfig' in 'AutoGenerator'
Cannot resolve constructor 'Builder(String, String, String, String)'
```

### 5.2 根本原因

MyBatis Plus Generator 3.5.x 版本中：
- `AutoGenerator` 构造函数变为私有
- `InjectionConfig` 被移除
- 配置全面改用 Builder/Fluent 模式

### 5.3 解决方案

使用 `FastAutoGenerator` 替代 `AutoGenerator`：

```java
// 修复前（旧版 API，不兼容）
public class CodeGenerator {
    public static void main(String[] args) {
        AutoGenerator mpg = new AutoGenerator(); // 错误：私有构造函数

        GlobalConfig gc = new GlobalConfig(); // 错误
        DataSourceConfig dsc = new DataSourceConfig(); // 错误
        InjectionConfig cfg = new InjectionConfig() {}; // 错误：已移除

        mpg.setGlobalConfig(gc);
        mpg.setDataSource(dsc);
        // ...
    }
}

// 修复后（新版 API）
public class CodeGenerator {
    public static void main(String[] args) {
        String projectPath = System.getProperty("user.dir");

        FastAutoGenerator.create(url, username, password)
                .globalConfig(builder -> {
                    builder.author(author)
                            .outputDir(projectPath + generateTo)
                            .disableOpenDir();
                })
                .packageConfig(builder -> {
                    builder.parent(parentPackage)
                            .moduleName(modelName)
                            .entity("model")
                            .pathInfo(Collections.singletonMap(OutputFile.xml, projectPath + mapperXmlPath));
                })
                .templateConfig(builder -> {
                    builder.disable(TemplateType.XML)
                            .mapper("/ftl/mapper.java");
                })
                .strategyConfig(builder -> {
                    builder.addInclude(scanner("表名, all全部表"))
                            .addTablePrefix(modelName + "_")
                            .entityBuilder()
                                .enableLombok()
                                .enableTableFieldAnnotation()
                                .naming(NamingStrategy.underline_to_camel)
                                .columnNaming(NamingStrategy.underline_to_camel)
                            .controllerBuilder()
                                .enableRestStyle()
                            .serviceBuilder()
                                .formatServiceFileName("%sService")
                                .formatServiceImplFileName("%sServiceImpl");
                })
                .templateEngine(new FreemarkerTemplateEngine())
                .execute();
    }
}
```

## 七、总结

本次迁移解决了以下关键问题：

1. **Security 配置类结构问题** - 清理了旧版残留代码，统一使用 Spring Security 6.x 配置方式
2. **Jakarta EE 全面迁移** - 注解相关包的完整替换
3. **第三方库 API 变更** - Commons Lang、Nacos RandomUtils 的标准库替代
4. **代码生成器升级** - 使用 MyBatis Plus 3.5.x 的 FastAutoGenerator

项目现已全面兼容 Spring Boot 3.x 技术栈，所有编译错误已消除。
