# 技术栈二次更迭遇到的问题3

> 日期：2026-05-10
> 范围：JDK 21 升级后运行时兼容性修复（JWT/SSO + Seata/Dubbo）

---

## 问题1：JWT 密钥长度不足导致登录失败

### 现象

用户登录时后端抛出异常：

```
io.jsonwebtoken.security.WeakKeyException: The specified key byte array is 64 bits
which is not secure enough for any JWT HMAC-SHA algorithm.
```

### 根因

JJWT 0.12.x（JDK 21 升级时引入）严格校验 HMAC 密钥长度。HS512 算法要求密钥 ≥ 64 字节（512 bits），而所有模块 `application.yml` 中配置的 `jwt.secret: mySecret` 仅 8 字节（64 bits），远低于最低要求。

旧版 JJWT 0.11.x 不做此校验，因此升级前不会报错。

### 修复

将全部 9 个模块的 `jwt.secret` 从 `mySecret`（8字节）替换为 69 字节的新密钥：

```yaml
# 修改前
secret: mySecret

# 修改后
secret: CooxiaoMall2026JwtSecretKeyForHs512AlgorithmMustBeAtLeast64BytesLong
```

涉及模块：mall-sso、mall-ams、mall-ums、mall-front、mall-order、mall-product、mall-search、mall-seckill、mall-resource

## 问题2：SSOFilter 提取 Token 带前导空格导致验证失败

### 现象

登录成功后立即显示"登录过期"，后端日志显示：
- Token 生成成功：`Token：eyJ...`（无空格）
- Token 验证失败：`JWT格式验证失败： eyJ...`（注意冒号后的空格）

### 根因

`SSOFilter` 提取 JWT token 的代码：

```java
String authToken = authHeader.substring(jwtTokenHead.length());
```

HTTP 标准 Authorization 头格式为 `"Bearer eyJhbGci..."`，截取 `"Bearer"`（7字符）后得到 `" eyJhbGci..."`（带前导空格）。`Jwts.parser().parseSignedClaims()` 对 Base64URL 输入严格校验，前导空格导致解析失败。

此 bug 之前就存在，但旧版 JJWT 对 token 格式宽容，不报错。0.12.x 版本严格校验后暴露。

### 修复

在所有 `substring(tokenHead.length())` 后添加 `.trim()`，共 12 处：

| 文件 | 说明 |
|------|------|
| 8 个 SSOFilter（sso/ams/order/front/product/ums/search/seckill） | 从 Authorization 头提取 token |
| AdminSSOServiceImpl.java:93 | 登出时提取 token |
| UserSSOServiceImpl.java:86 | 登出时提取 token |
| UserInfoServiceImpl.java:30 | 获取用户信息时提取 token |
| UserController.java:90 | 修改密码时提取 token |

```java
// 修改前
String authToken = authHeader.substring(jwtTokenHead.length());

// 修改后
String authToken = authHeader.substring(jwtTokenHead.length()).trim();
```

## 问题3：Seata + Dubbo 集成 NPE（branchType is null）

### 现象

秒杀模块浏览商品时报错：

```
Cannot invoke "org.apache.seata.core.model.BranchType.name()" because "branchType" is null
```

Dubbo RPC 调用完全失败，重试 3 次后抛出异常。

### 根因

这是 Apache Seata 2.1.0 的已知 bug（GitHub Issue #6815）。

`dubbo-filter-seata:1.0.2`（由 `seata-all:2.1.0` 传递引入）的消费者端过滤器代码：

```java
// SeataTransactionPropagationConsumerFilter.invoke()
String xid = RootContext.getXID();               // 始终调用
BranchType branchType = RootContext.getBranchType();  // 始终调用，即使不在事务中
if (xid != null) { ... }                         // 仅检查 xid
```

桥接类 `io.seata.core.context.RootContext.convertIoSeata()` 在 `branchType` 为 null 时直接调用 `branchType.name()` 导致 NPE：

```java
// seata-all-2.1.0.jar 中的 bug
private static io.seata.core.model.BranchType convertIoSeata(
        org.apache.seata.core.model.BranchType branchType) {
    return io.seata.core.model.BranchType.get(branchType.name()); // NPE!
}
```

当 Dubbo 调用发生在 `@GlobalTransactional` 作用域之外（如 Controller 调用 Dubbo 服务，而只有服务实现类有事务注解），`RootContext` 中没有 XID，`getBranchType()` 返回 null，触发 NPE。

### 修复

1. **排除有 bug 的依赖**：在根 pom.xml 的 `seata-spring-boot-starter` 声明中排除 `dubbo-filter-seata:1.0.2`

2. **创建自定义 Dubbo Filter** 替代，正确处理 null 情况：
   - `SeataTransactionConsumerFilter`：消费者端，仅在 xid 非空时传播，且 branchType 做 null 检查
   - `SeataTransactionProviderFilter`：提供者端，绑定/解绑 XID 和 branchType，支持 AT 和 TCC 模式

3. **通过 Dubbo SPI 注册**：在 `META-INF/dubbo/org.apache.dubbo.rpc.Filter` 中声明

关键代码对比：

```java
// 旧版（有 bug）- 无条件调用 getBranchType()
BranchType branchType = RootContext.getBranchType(); // NPE when not in tx
if (xid != null) {
    invocation.setAttachment("TX_XID", xid);
    if (branchType != null) { ... }
}

// 新版（修复）- 先检查 xid，再安全获取 branchType
String xid = RootContext.getXID();
if (xid != null) {
    invocation.setAttachment("TX_XID", xid);
    BranchType branchType = RootContext.getBranchType();
    if (branchType != null) {
        invocation.setAttachment("TX_BRANCH_TYPE", branchType.name());
    }
}
```

涉及文件：
- `pom.xml`（根）：排除 dubbo-filter-seata
- `mall-common/pom.xml`：新增 dubbo 和 seata-all 的 provided 依赖
- `mall-common/src/main/java/com/cooxiao/mall/common/dubbo/SeataTransactionConsumerFilter.java`（新增）
- `mall-common/src/main/java/com/cooxiao/mall/common/dubbo/SeataTransactionProviderFilter.java`（新增）
- `mall-common/src/main/resources/META-INF/dubbo/org.apache.dubbo.rpc.Filter`（新增）
