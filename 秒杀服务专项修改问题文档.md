## 一、布隆过滤器不可用（RedisBloom 模块缺失）

### 问题描述
`SeckillBloomInitialJob` 使用 `bf.madd`、`bf.exists` 等 RedisBloom 模块命令，但本地开发环境的 Redis 3.2.100 (Windows) 不支持该模块，导致：
- 布隆过滤器 key 无法创建
- `SeckillSkuServiceImpl.listSeckillSkus()` 报错 "布隆过滤器未创建"

### 解决方案
- **开发阶段**：用 Redis Set 替代布隆过滤器（`SADD` / `SISMEMBER`），所有 Redis 版本均支持
- **生产部署**：在 Linux 服务器安装 RedisBloom 模块后，切换回布隆过滤器实现
- 代码中留有 `TODO` 注释标记切换点

### 修改文件
- `SeckillBloomInitialJob.java`：`RedisBloomUtils.bfmadd()` → `RedisTemplate.boundSetOps().add()`
- `SeckillSkuServiceImpl.java`：`RedisBloomUtils.bfexists()` → `RedisTemplate.boundSetOps().isMember()`

## 二、秒杀订单创建缓慢 + 控制台日志刷屏

### 问题描述
创建秒杀订单时响应非常慢（卡顿数秒），控制台疯狂输出 DEBUG 日志导致 IDE 卡死。

### 根因分析
三层问题叠加：
1. `success` 表结构（`user_phone` 字段不存在）导致 INSERT 失败
2. RabbitMQ 消费者 `basicNack(requeue=true)` 导致失败消息无限重试
3. `logging.level.com.cooxiao.mall: trace` 日志级别过低，每次重试输出大量 DEBUG 信息

### 解决方案
1. 修复 `success` 表结构，使其与实体类对齐
2. 消费者异常时抛出 RuntimeException，让 `@Transactional` 回滚事务
3. 日志级别从 `trace` 改为 `info`

### 修改文件
- `05-seckill.sql`：重建 `success` 表，添加 `sku_id`、`title`、`main_picture`、`seckill_price`、`quantity`、`bar_code`、`data`、`order_sn` 等字段，`phone` → `user_phone`，`seckill_sku_id` → `seckill_id`
- `SeckillQueueConsumer.java`：异常时抛出 RuntimeException 触发事务回滚
- `application-test.yml`：`logging.level.com.cooxiao.mall: trace` → `info`

## 三、秒杀库存变负数（超卖问题）

### 问题描述
`seckill_sku` 表中 `seckill_stock` 出现负数（-15937），原因是：
1. 消费者无限重试时，`updateReduceStockBySkuId` 每次都执行成功（事务未回滚）
2. 扣减库存 SQL 没有防超卖条件

### 解决方案
1. SQL 加 `AND seckill_stock >= #{quantity}` 防止库存扣为负数
2. 消费者检查返回影响行数，为 0 时说明库存不足
3. 修复事务回滚机制，防止重复扣减

### 修改文件
- `SeckillSkuMapper.xml`：`WHERE sku_id=#{skuId}` → `WHERE sku_id=#{skuId} AND seckill_stock >= #{quantity}`
- `SeckillQueueConsumer.java`：检查 `updateReduceStockBySkuId` 返回值

## 四、success 表 id 字段不能为空

### 问题描述
`saveSuccess` 是自定义 XML INSERT 语句，不经过 MyBatis-Plus 的 `ASSIGN_ID` 生成逻辑，导致 `id` 为 null，数据库报 `Column 'id' cannot be null`。

### 解决方案
1. `SuccessMapper.xml` 中 INSERT 语句加上 `id` 列和 `#{id}` 参数
2. `SeckillServiceImpl` 中创建 Success 对象时手动调用 `IdWorker.getId()` 生成雪花 ID
3. `SeckillQueueConsumer` 中增加 null 检查，兼容旧消息

### 修改文件
- `SuccessMapper.xml`：INSERT 增加 `id` 列
- `SeckillServiceImpl.java`：`success.setId(IdWorker.getId())`
- `SeckillQueueConsumer.java`：`if(success.getId() == null) success.setId(IdWorker.getId())`
