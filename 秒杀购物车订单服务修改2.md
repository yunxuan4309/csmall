## 一、实体类字段映射问题修复

### 1. OmsCart 实体类字段映射修复

**问题描述：**
- 数据库表 `oms_cart` 中图片列名是 `picture_url`
- 实体类 `OmsCart` 中字段名是 `mainPicture`
- MyBatis-Plus 默认将 `mainPicture` 映射为 `main_picture`，导致 SQL 查询报错

**错误信息：**
```
java.sql.SQLSyntaxErrorException: Unknown column 'main_picture' in 'field list'
```

**解决方案：**
在 `OmsCart.java` 中添加 `@TableField` 注解：

```java
@TableField("picture_url")
private String mainPicture;
```

**文件路径：** `mall-pojo/src/main/java/com/cooxiao/mall/pojo/order/model/OmsCart.java`

### 2. OmsOrderItem 实体类字段映射修复

**问题描述：**
- 数据库列 `sku_properties` 与实体字段 `data` 不匹配
- 数据库列 `picture_url` 与实体字段 `mainPicture` 不匹配
- 缺少 `total_price` 和 `spu_name` 字段

**解决方案：**
在 `OmsOrderItem.java` 中：

```java
@TableField("sku_properties")
private String data;

@TableField("picture_url")
private String mainPicture;

// 新增字段
private BigDecimal totalPrice;  // 对应 total_price
private String spuName;          // 对应 spu_name
```

**文件路径：** `mall-pojo/src/main/java/com/cooxiao/mall/pojo/order/model/OmsOrderItem.java`

## 二、分类数据与产品数据不匹配问题

### 问题描述

前端点击"男装"分类时，显示的是电子产品。

**问题根源：**
- 数据库 `pms_category` 表中 id=6 是"男装"
- 但 `pms_spu` 表中男装产品的 `category_id=8`
- 导致查询 category_id=6 时返回了错误的产品

### 解决方案

重新执行测试数据初始化脚本，确保分类表数据与产品表外键一致：

```sql
USE cs_mall_pms;

-- 清空并重新插入分类数据
DELETE FROM pms_category;
INSERT INTO `pms_category` (`id`, `name`, `parent_id`, `depth`, `sort`, `enable`, `is_parent`, `is_display`) VALUES 
(1, '手机数码', 0, 1, 1, 1, 1, 1),
(2, '电脑办公', 0, 1, 2, 1, 1, 1),
(3, '服装鞋帽', 0, 1, 3, 1, 1, 1),
(4, '手机', 1, 2, 1, 1, 0, 1),
(5, '平板电脑', 1, 2, 2, 1, 0, 1),
(6, '笔记本', 2, 2, 1, 1, 0, 1),
(7, '台式机', 2, 2, 2, 1, 0, 1),
(8, '男装', 3, 2, 1, 1, 0, 1),
(9, '女装', 3, 2, 2, 1, 0, 1);
```

**附加操作：** 删除 Redis 中的 `category_tree` 缓存后刷新页面。

## 五、经验总结

1. **实体类字段命名规范**：当实体类字段名与数据库列名不一致时，必须使用 `@TableField` 注解显式指定映射关系。

2. **测试数据一致性**：初始化测试数据时，确保外键关联数据的一致性，分类表的 id 必须与产品表的 category_id 对应。

3. **Redis 缓存问题排查**：当数据不一致时，注意检查 Redis 缓存是否存储了旧数据。

4. **问题排查流程**：
   - 前端报错 → 查看后端日志 → 定位 SQL 错误
   - 对比数据库表结构与实体类字段
   - 使用 `@TableField` 解决字段映射问题
