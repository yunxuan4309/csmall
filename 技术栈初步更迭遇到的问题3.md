## 一、MyBatis-Plus 分页类型不兼容问题

### 1.1 问题描述

```
不兼容的类型: com.baomidou.mybatisplus.core.metadata.IPage无法转换为com.baomidou.mybatisplus.extension.plugins.pagination.Page
推论变量T具有不兼容的等式约束条件
```

### 1.2 根本原因

使用 MyBatis-Plus 的 `selectPage()` 方法时：
- `selectPage(Page<T>, Wrapper<T>)` 返回 `IPage<T>`
- 但原有代码直接返回 `IPage<Model>` 给需要 `IPage<VO>` 的方法
- `JsonPage.restPage()` 需要泛型类型完全匹配

### 1.3 解决方案

将分页结果从 Model 类型转换为 VO 类型：

```java
// 修复前（错误）
@Override
public JsonPage<AlbumStandardVO> list(Integer page, Integer pageSize) {
    Page<Album> pageParam = new Page<>(page, pageSize);
    LambdaQueryWrapper<Album> wrapper = new LambdaQueryWrapper<>();
    wrapper.orderByDesc(Album::getSort, Album::getGmtCreate);
    IPage<Album> result = albumMapper.selectPage(pageParam, wrapper);
    return JsonPage.restPage(result); // 错误：Album 不是 AlbumStandardVO
}

// 修复后（正确）
@Override
public JsonPage<AlbumStandardVO> list(Integer page, Integer pageSize) {
    Page<Album> pageParam = new Page<>(page, pageSize);
    LambdaQueryWrapper<Album> wrapper = new LambdaQueryWrapper<>();
    wrapper.orderByDesc(Album::getSort, Album::getGmtCreate);
    IPage<Album> result = albumMapper.selectPage(pageParam, wrapper);

    // Model -> VO 类型转换
    List<AlbumStandardVO> voList = result.getRecords().stream()
            .map(this::convertToVO)
            .collect(Collectors.toList());

    // 构建新的分页结果
    IPage<AlbumStandardVO> pageVO = new Page<>(
            result.getCurrent(), result.getSize(), result.getTotal());
    pageVO.setRecords(voList);
    return JsonPage.restPage(pageVO);
}

private AlbumStandardVO convertToVO(Album album) {
    if (album == null) {
        return null;
    }
    AlbumStandardVO vo = new AlbumStandardVO();
    BeanUtils.copyProperties(album, vo);
    return vo;
}
```

## 二、Mapper 方法缺失问题

### 2.1 问题描述

```
找不到符号 selectPage
找不到符号 updateFullInfoById
```

### 2.2 根本原因

Mapper 接口继承了 `BaseMapper<T>` 但某些自定义方法在接口中未声明，或调用方使用方法签名与声明不一致。

### 2.3 解决方案

**方案一**：在 Mapper 接口中添加方法声明

```java
// BrandMapper.java
@Repository
public interface BrandMapper extends BaseMapper<Brand> {

    // 添加缺失的方法声明
    int updateFullInfoById(Brand brand);
}
```

**方案二**：对于分页查询，使用 `LambdaQueryWrapper` 替代自定义分页方法

```java
// 修复前（调用不存在的分页方法）
@Override
public JsonPage<BrandStandardVO> listByCategoryId(Long categoryId, Integer page, Integer pageSize) {
    return brandMapper.listByCategoryId(categoryId, page, pageSize); // 方法不存在
}

// 修复后（使用 LambdaQueryWrapper）
@Override
public JsonPage<BrandStandardVO> listByCategoryId(Long categoryId, Integer page, Integer pageSize) {
    Page<Brand> pageParam = new Page<>(page, pageSize);
    LambdaQueryWrapper<Brand> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(Brand::getId, categoryId)
           .orderByDesc(Brand::getSort, Brand::getGmtCreate);
    IPage<Brand> result = brandMapper.selectPage(pageParam, wrapper);

    List<BrandStandardVO> voList = result.getRecords().stream()
            .map(this::convertToVO)
            .collect(Collectors.toList());

    IPage<BrandStandardVO> pageVO = new Page<>(
            result.getCurrent(), result.getSize(), result.getTotal());
    pageVO.setRecords(voList);
    return JsonPage.restPage(pageVO);
}
```

## 三、update() 方法不存在问题

### 3.1 问题描述

```
不兼容的类型: com.cooxiao.mall.pojo.product.model.Album无法转换为com.baomidou.mybatisplus.core.conditions.Wrapper<com.cooxiao.mall.pojo.product.model.Album>
```

### 3.2 根本原因

MyBatis-Plus 的 `BaseMapper` 没有 `update(T entity)` 方法，只有 `updateById(T entity)` 或 `update(T entity, Wrapper<T> wrapper)`。

### 3.3 解决方案

将 `mapper.update(entity)` 替换为 `mapper.updateById(entity)`：

```java
// 修复前（错误）
int rows = albumMapper.update(album);

// 修复后（正确）
int rows = albumMapper.updateById(album);
```

### 3.4 修改文件列表

| 模块 | 文件 | 修改内容 |
|------|------|----------|
| mall-product | `AlbumServiceImpl.java` | `update()` → `updateById()` |
| mall-product | `AttributeServiceImpl.java` | `update()` → `updateById()` |
| mall-product | `AttributeTemplateServiceImpl.java` | `update()` → `updateById()` |

## 四、类型转换兼容性问题

### 4.1 问题描述

```
不兼容的类型: java.lang.Long无法转换为int
```

### 4.2 根本原因

MyBatis-Plus 3.x 中 `IService.selectCount()` 返回 `long` 类型，而不是 `int`。

### 4.3 解决方案

将 `int` 改为 `long`：

```java
// 修复前（错误）
int count = deliveryAddressMapper.selectCount(wrapper);

// 修复后（正确）
long count = deliveryAddressMapper.selectCount(wrapper);
```

### 4.4 修改文件列表

| 模块 | 文件 | 修改内容 |
|------|------|----------|
| mall-ums | `DeliveryAddressServiceImpl.java` | `int count` → `long count` |

## 六、关键代码模式

### 6.1 分页查询标准模板

```java
@Override
public JsonPage<XXXVO> list(Integer page, Integer pageSize) {
    Page<XXX> pageParam = new Page<>(page, pageSize);
    LambdaQueryWrapper<XXX> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(XXX::getField, value)
           .orderByDesc(XXX::getSort, XXX::getGmtCreate);
    IPage<XXX> result = mapper.selectPage(pageParam, wrapper);

    // 转换为VO
    List<XXXVO> voList = result.getRecords().stream()
            .map(this::convertToVO)
            .collect(Collectors.toList());

    // 构建分页结果
    IPage<XXXVO> pageVO = new Page<>(
            result.getCurrent(), result.getSize(), result.getTotal());
    pageVO.setRecords(voList);
    return JsonPage.restPage(pageVO);
}

private XXXVO convertToVO(XXX model) {
    if (model == null) {
        return null;
    }
    XXXVO vo = new XXXVO();
    BeanUtils.copyProperties(model, vo);
    return vo;
}
```

## 七、总结

本次迁移解决了以下关键问题：

1. **MyBatis-Plus 分页类型不兼容** - 使用 stream + convertToVO 模式将 `IPage<Model>` 转换为 `IPage<VO>`
2. **Mapper 方法缺失** - 在接口中添加方法声明，或使用 `LambdaQueryWrapper.selectPage()` 替代
3. **update() 方法替换** - 使用 `updateById()` 替代不存在的 `update()`
4. **类型转换兼容性** - `selectCount()` 返回 `long`，需要相应调整接收变量类型

项目现已全面兼容 Spring Boot 3.x 和 MyBatis-Plus 3.x，所有编译错误已消除。
