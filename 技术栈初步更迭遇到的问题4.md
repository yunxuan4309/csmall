# Spring Boot 2.x → 3.x 技术栈迁移问题总结（第四轮）

## 问题背景

在完成前三轮迁移后，项目启动时发现 mall-gateway-server 模块存在 Spring Cloud Gateway 与 Knife4j 的循环依赖问题，导致应用无法启动。

---

## 一、Gateway + Knife4j 循环依赖问题

### 1.1 问题描述

```
***************************
APPLICATION FAILED TO START
***************************

Description:

The dependencies of some of the beans in the application context form a cycle:

┌─────┐
|  knife4jSwaggerProvider (field private org.springframework.cloud.gateway.route.RouteLocator com.cooxiao.mall.gateway.knife4j.config.Knife4jSwaggerProvider.routeLocator)
↑     ↓
|  cachedCompositeRouteLocator defined in class path resource [org/springframework/cloud/gateway/config/GatewayAutoConfiguration.class]
↑     ↓
|  routeDefinitionRouteLocator defined in class path resource [org/springframework/cloud/gateway/config/GatewayAutoConfiguration.class]
↑     ↓
|  modifyRequestBodyGatewayFilterFactory defined in class path resource [org/springframework/cloud/gateway/config/GatewayAutoConfiguration.class]
↑     ↓
|  org.springframework.boot.autoconfigure.web.reactive.WebFluxAutoConfiguration$EnableWebFluxConfiguration
└─────┘
```

### 1.2 根本原因

`Knife4jSwaggerProvider` 类在实现 `WebFluxConfigurer` 接口时，直接使用 `@Autowired` 注入了 `RouteLocator`：

```java
@Component
public class Knife4jSwaggerProvider implements WebFluxConfigurer {

    @Autowired
    private RouteLocator routeLocator;  // 导致循环依赖
}
```

循环依赖链路：
```
Knife4jSwaggerProvider (WebFluxConfigurer)
    ↓ @Autowired RouteLocator
cachedCompositeRouteLocator (RouteLocator)
    ↓ 依赖
routeDefinitionRouteLocator
    ↓ 依赖
modifyRequestBodyGatewayFilterFactory
    ↓ 依赖
EnableWebFluxConfiguration (WebFluxConfigurer)
    ↓ 循环!
Knife4jSwaggerProvider
```

### 1.3 解决方案

采用 **`ObjectProvider` 构造器注入**方式，从根本上避免循环依赖：

```java
// 修复前（字段注入，导致循环依赖）
@Autowired
private RouteLocator routeLocator;

// 修复后（构造器注入 ObjectProvider）
private final ObjectProvider<RouteLocator> routeLocatorProvider;

public Knife4jSwaggerProvider(ObjectProvider<RouteLocator> routeLocatorProvider) {
    this.routeLocatorProvider = routeLocatorProvider;
}
```

并在获取路由时添加空值检查：

```java
public List<SwaggerResource> getSwaggerResources() {
    // ...
    RouteLocator routeLocator = routeLocatorProvider.getIfAvailable();
    if (routeLocator == null) {
        return resources;
    }
    // ...
}
```

### 1.4 修改文件列表

| 模块 | 文件 | 修改内容 |
|------|------|----------|
| mall-gateway-server | `Knife4jSwaggerProvider.java` | 字段注入改为构造器注入 ObjectProvider |

### 1.5 修改详情

**文件**: `mall-gateway-server/src/main/java/com/cooxiao/mall/gateway/knife4j/config/Knife4jSwaggerProvider.java`

| 变更点 | 旧代码 | 新代码 |
|--------|--------|--------|
| 导入 | `import org.springframework.beans.factory.annotation.Autowired;` | `import org.springframework.beans.factory.ObjectProvider;` |
| 字段 | `@Autowired private RouteLocator routeLocator;` | `private final ObjectProvider<RouteLocator> routeLocatorProvider;` |
| 构造器 | 无 | `public Knife4jSwaggerProvider(ObjectProvider<RouteLocator> routeLocatorProvider)` |
| 路由获取 | 直接使用 `routeLocator` | `routeLocatorProvider.getIfAvailable()` + 空值检查 |

## 三、关键代码模式

### 3.1 Gateway SwaggerProvider 修复模板

```java
package com.cooxiao.mall.gateway.xxx.config;

import org.springframework.beans.factory.ObjectProvider;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.cloud.gateway.route.RouteLocator;
import org.springframework.stereotype.Component;
import org.springframework.web.reactive.config.ResourceHandlerRegistry;
import org.springframework.web.reactive.config.WebFluxConfigurer;

@Component
public class Knife4jSwaggerProvider implements WebFluxConfigurer {

    public static final String API_URI = "/v3/api-docs";

    // 使用 ObjectProvider 延迟获取，避免循环依赖
    private final ObjectProvider<RouteLocator> routeLocatorProvider;

    @Value("${spring.application.name}")
    private String applicationName;

    // 构造器注入
    public Knife4jSwaggerProvider(ObjectProvider<RouteLocator> routeLocatorProvider) {
        this.routeLocatorProvider = routeLocatorProvider;
    }

    public List<SwaggerResource> getSwaggerResources() {
        List<SwaggerResource> resources = new ArrayList<>();
        List<String> routeHosts = new ArrayList<>();

        // 关键：使用 getIfAvailable() 延迟获取
        RouteLocator routeLocator = routeLocatorProvider.getIfAvailable();
        if (routeLocator == null) {
            return resources;
        }

        routeLocator.getRoutes()
                .filter(route -> route.getUri().getHost() != null)
                .filter(route -> !applicationName.equals(route.getUri().getHost()))
                .subscribe(route -> routeHosts.add(route.getUri().getHost()));
        // ... 后续逻辑
        return resources;
    }

    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        registry.addResourceHandler("/doc.html")
                .addResourceLocations("classpath:/META-INF/resources/");
        registry.addResourceHandler("/webjars/**")
                .addResourceLocations("classpath:/META-INF/resources/webjars/");
    }
}
```

### 3.2 循环依赖解决方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| `@Lazy` 注解 | 改动最小 | 只是延迟加载，可靠性稍差 | 快速修复 |
| `ObjectProvider` 构造器注入 | 根本解决循环依赖 | 需要重构代码 | 正式项目（推荐） |
| `ApplicationContextAware` | 运行时获取 | 侵入性较大 | 特殊场景 |

## 五、总结

本次迁移解决了以下关键问题：

1. **Gateway + Knife4j 循环依赖** - 使用 `ObjectProvider<RouteLocator>` 构造器注入替代字段注入

项目现已全面兼容 Spring Boot 3.x 和 MyBatis-Plus 3.x，所有已知的启动问题已消除。
