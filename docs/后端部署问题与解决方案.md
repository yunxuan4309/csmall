# 后端部署问题与解决方案

> 部署时间：2026-05-11
> 服务器：阿里云 ECS（成都），4 vCPU 16 GiB，Alibaba Cloud Linux 3
> 项目：CoolShark 微服务电商平台

---

## 一、中间件安装问题

### 1. RabbitMQ Erlang 版本冲突

**问题**：RabbitMQ 3.13.7 要求 Erlang >= 26.0，但 yum 默认源只有 Erlang 22。packagecloud.io 的 Erlang 仓库脚本不识别 Alibaba Cloud Linux 3（alinux/3），返回 404。

**解决**：
```bash
# 用 os=el dist=8 参数伪装为 RHEL 8
curl -s https://packagecloud.io/install/repositories/rabbitmq/erlang/script.rpm.sh | sudo os=el dist=8 bash
# 包名是 erlang 而不是 erlang-26
yum install -y erlang
```

安装 RabbitMQ 时，旧版 Erlang 22（erlang-erts-22.0.7）与新版 Erlang 26 文件冲突，需先卸载旧版：
```bash
sudo yum remove -y erlang-erts
sudo yum install -y https://github.com/rabbitmq/rabbitmq-server/releases/download/v3.13.7/rabbitmq-server-3.13.7-1.el8.noarch.rpm
```

### 2. Seata Server 启动失败 - 缺少 secretKey

**问题**：Seata 2.1.0 启动报错 `Could not resolve placeholder 'seata.security.secretKey'`

**解决**：在 `application.yml` 中添加：
```yaml
seata:
  security:
    secretKey: <你的密钥>
    tokenValidityInMilliseconds: 1800000
```

### 3. Seata Server 启动参数 -d 不支持

**问题**：`seata-server.sh -d` 报 `Unknown argument: -d`

**解决**：Seata 2.x 不支持 `-d` 守护进程参数，用 nohup 后台启动：
```bash
sudo nohup sh /data/seata/bin/seata-server.sh > /data/seata/logs/start.log 2>&1 &
```

### 4. Seata 下载文件名变化

**问题**：旧版下载地址 `seata-server-2.1.0.tar.gz` 返回 404

**解决**：Apache 孵化后文件名变为 `apache-seata-2.1.0-incubating-bin.tar.gz`，下载地址：
```
https://archive.apache.org/dist/incubator/seata/2.1.0/apache-seata-2.1.0-incubating-bin.tar.gz
```
解压后目录名为 `apache-seata-2.1.0-incubating-bin`（注意有 `-bin` 后缀）。

### 5. Elasticsearch IK 中文分词插件

**问题**：mall-search 启动报错 `analyzer [ik_max_word] has not been configured in mappings`

**解决**：ES 必须安装 IK 分词插件，且版本需与 ES 版本一致：
```bash
sudo /usr/share/elasticsearch/bin/elasticsearch-plugin install https://release.infinilabs.com/analysis-ik/stable/elasticsearch-analysis-ik-7.17.29.zip
sudo systemctl restart elasticsearch
```
注意：IK 插件已从 GitHub `medcl/elasticsearch-analysis-ik` 迁移到 infinilabs 的发布站。

### 6. MySQL my.cnf 配置格式错误

**问题**：直接在 `/etc/my.cnf` 顶部添加配置行，没有 `[mysqld]` 段头，导致 MySQL 启动失败：`Found option without preceding group in config file`

**解决**：不在 `/etc/my.cnf` 直接修改，而是创建独立配置文件 `/etc/my.cnf.d/csmall.cnf`：
```ini
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_general_ci
```

### 7. MySQL lower_case_table_names 不可事后修改

**问题**：MySQL 8.0 初始化后不能修改 `lower_case_table_names`，修改后启动失败

**解决**：删除该配置项，保持默认值 0（区分大小写）。如果确实需要不区分大小写，必须在首次初始化前设置。

## 二、应用部署问题

### 8. Maven 打包失败 - 纯库模块被 spring-boot-maven-plugin repackage

**问题**：`mall-front-service`、`mall-ams-service` 等纯库模块没有 main 方法，但继承了父 pom 的 `spring-boot-maven-plugin`，导致 repackage 失败：`Unable to find main class`

**解决**：在父 pom 中设置 `skip=true`，在每个有 main 方法的 webapi 模块中覆盖为 `skip=false`：

父 pom：
```xml
<plugin>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-maven-plugin</artifactId>
    <configuration>
        <skip>true</skip>
    </configuration>
</plugin>
```

webapi 模块：
```xml
<plugin>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-maven-plugin</artifactId>
    <configuration>
        <skip>false</skip>
    </configuration>
</plugin>
```

### 9. mall-product 启动失败 - 缺少文件上传路径配置

**问题**：`Could not resolve placeholder 'custom.file-upload.server-local-path'`

**解决**：prod 配置文件中补充：
```yaml
custom:
  file-upload:
    server-local-path: /data/csmall-upload
    resource-host: ${RESOURCE_HOST:http://8.156.85.160/}
```

### 10. 环境变量未传递导致 MySQL 认证失败

**问题**：手动用 nohup 启动服务时没有 export 环境变量，MySQL 连接使用了默认值 `root/root`，认证失败

**解决**：启动服务前必须 export 所有环境变量，或使用包含 export 的启动脚本 `start.sh`

## 三、网络与安全组问题

### 11. 外网无法访问 80 端口

**问题**：浏览器访问 `http://8.156.85.160/` 超时

**原因**：阿里云安全组默认只开放了 SSH(22) 和 RDP(3389)，没有开放 HTTP(80)

**解决**：在安全组入方向规则中添加 TCP 80/80 端口，授权对象 0.0.0.0/0

### 12. ICP 备案需服务器至少 3 个月

**问题**：域名备案时提示"云服务可用性 未找到账号下可备案的服务器/服务码"

**原因**：阿里云要求服务器至少购买 3 个月才能用于 ICP 备案

**解决**：采用方案 A，先通过 IP 地址访问，后续续费后再进行备案

## 四、其他注意事项

### Git 中文乱码

Windows CMD 中 git log 显示中文乱码，需执行：
```bash
git config --global core.quotepath false
git config --global i18n.commitEncoding utf-8
git config --global i18n.logOutputEncoding utf-8
git config --global gui.encoding utf-8
```

### scp 上传目录不存在

scp 上传到服务器时目标目录不存在会报 `dest open: Failure`，需先在服务器上 `mkdir -p` 创建目录。
