# 安卓端与服务器端图片上传与查看问题解决记录

> 问题核心：带 Token 上传图片后，服务器端线程永久挂死，导致安卓端拍照上传和查看历史接口全部超时。

---

## 一、问题链路总览

```
安卓端  →  Nginx(:80)  →  Gateway(:10087)  →  Resource(:9060)  →  MySQL(cs_mall_resource)
                                                                     ↑
                                                           此处线程挂死，请求无法返回
```

---

## 二、排查全过程

### 阶段一：上传返回 503

**现象**：Android App 拍照上传返回 `503 Service Unavailable`

**排查**：
1. `systemctl status mall-resource` → 服务正常运行（active）
2. `curl 127.0.0.1:8848/nacos/v1/cs/configs...` → mall-resource 未注册到 Nacos
3. 检查 `application-prod.yml` → Gateway 路由使用 `lb://mall-resource`

**根因**：mall-resource 的 pom.xml 没有 `spring-cloud-starter-alibaba-nacos-discovery`，不会注册到 Nacos，Gateway 的 `lb://` 找不到目标。

**解决**：Gateway 路由 `uri` 从 `lb://mall-resource` 改为 `http://127.0.0.1:9060`（直连）。

---

### 阶段二：上传超时（504 / 499）

**现象**：503 修复后，Android 拍照上传变为超时。Nginx 日志：
- 小图（189KB）成功：`POST /upload/picture/single → 200`
- 大图超时：`POST /upload/picture/single → 499`（客户端关闭连接）
- 后端返回无响应时：`→ 504`（Nginx 等待后端 60 秒超时）

**排查**：
1. 服务器 `time curl 127.0.0.1:10087/upload/` 不带 Token → 221ms 正常
2. 服务器 `time curl 8.156.85.160/upload/` 不带 Token → 36ms 正常
3. 服务器 `time curl 8.156.85.160/upload/` **带 Token** → **卡住不动**
4. 对比差异：带 Token 才会触发 `saveRecord()` 写入数据库

**根因**：带 Token 的请求走了数据库写入路径，Druid 连接池 `max-wait=0`（默认无限等待），获取数据库连接时线程永久挂死，不抛异常，try-catch 也无法捕获。

---

### 阶段三：Druid 连接池配置无效

**现象**：在 `application-prod.yml` 中添加了 `spring.datasource.druid.max-wait: 10000` 后仍然超时。

**排查**：
1. 重新打包部署后带 Token 测试 → 仍然卡住
2. 检查 pom.xml → 使用的是 `com.alibaba:druid`（普通版）
3. `spring.datasource.druid.*` 命名空间仅在 `druid-spring-boot-starter` 中生效

**根因**：`com.alibaba:druid` 不支持 Spring Boot 配置命名空间，加的所有 `druid.*` 配置均未生效。Druid 使用默认配置，`max-wait=0` 继续生效。

**解决**：去掉 Druid，改用 Spring Boot 自带的 HikariCP 连接池：
- 删除 `com.alibaba:druid` 依赖
- 删除 `spring.datasource.type: com.alibaba.druid.pool.DruidDataSource`
- 改用 `spring.datasource.hikari.connection-timeout: 10000`（HikariCP 原生支持）

---

### 阶段四：数据库权限缺失

**现象**：换成 HikariCP 后上传很快成功（123ms），但查询历史接口返回 500：
```
Failed to obtain JDBC Connection
```

**排查**：
1. `csmall.env` 中 DB 用户为 `csmall_app`，密码 `Csmall2026!App`
2. `SHOW GRANTS FOR 'csmall_app'@'%'` → 仅有 `cs_mall_ams/oms/pms/seckill/ums` 权限，**没有 `cs_mall_resource`**

**根因**：`cs_mall_resource` 是本次新增数据库，创建后未对 `csmall_app` 授权。

**解决**：`GRANT ALL PRIVILEGES ON cs_mall_resource.* TO 'csmall_app'@'%'`

## 五、经验教训

1. **连接池超时配置必须验证生效性**：写了配置不代表生效，特别是第三方库的 Spring Boot 自动配置兼容性问题。Druid 普通版和 Starter 版配置方式不同。
2. **Spring Boot 自带连接池（HikariCP）是最可靠的选择**：无需额外依赖，配置即生效。
3. **新增数据库要同步授权**：每次创建新数据库，记得对 `csmall_app` 用户授权。
4. **排查方法**：从外到内逐层测试——本地 curl → Nginx curl → Gateway curl → 服务直连，快速定位问题层。
