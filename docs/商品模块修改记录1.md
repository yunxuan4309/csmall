## 一、mall-product 模块编译错误 — MyBatis-Plus 依赖缺失

### 问题描述
`mall-product-webapi` 中大量类引用了 `com.baomidou.mybatisplus.*` 包，但 `pom.xml` 中只有 `mybatis-spring-boot-starter`，缺少 MyBatis-Plus 依赖，导致编译报错：
```
java: 程序包com.baomidou.mybatisplus.extension.plugins.pagination不存在
```

### 解决方案
在 `mall-product/mall-product-webapi/pom.xml` 中添加 MyBatis-Plus 依赖：

```xml
<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-spring-boot3-starter</artifactId>
</dependency>
<dependency>
    <groupId>com.baomidou</groupId>
    <artifactId>mybatis-plus-jsqlparser</artifactId>
    <version>${mybatis-plus-jsqlparser.version}</version>
</dependency>
```

> 注意：Spring Boot 3.x 必须使用 `mybatis-plus-spring-boot3-starter`，不能使用旧版 `mybatis-plus-boot-starter`。

## 三、全模块实体类缺少 @TableName 注解

### 问题描述
所有 `mall-pojo` 中的商品、用户、订单、秒杀实体类都没有 `@TableName` 注解。MyBatis-Plus 的 `BaseMapper` 自动生成 SQL 时，默认用类名转蛇形作为表名，导致表名错误：

| 实体类 | 默认表名(错误) | 实际表名 |
|---|---|---|
| `Spu` | `spu` | `pms_spu` |
| `Category` | `category` | `pms_category` |
| `DeliveryAddress` | `delivery_address` | `ums_delivery_address` |
| ... | ... | ... |

运行时报错：
```
Table 'cs_mall_pms.spu' doesn't exist
```

### 解决方案
为所有实体类添加 `@TableName` 注解，共修改 25 个文件：

**商品模块（13个）：**
- `Album` → `@TableName("pms_album")`
- `Attribute` → `@TableName("pms_attribute")`
- `AttributeTemplate` → `@TableName("pms_attribute_template")`
- `AttributeTemplateValue` → `@TableName("pms_attribute_template_value")`
- `Brand` → `@TableName("pms_brand")`
- `BrandCategory` → `@TableName("pms_brand_category")`
- `Category` → `@TableName("pms_category")`
- `CategoryAttributeTemplate` → `@TableName("pms_category_attribute_template")`
- `Picture` → `@TableName("pms_picture")`
- `Sku` → `@TableName("pms_sku")`
- `SkuSpecification` → `@TableName("pms_sku_specification")`
- `Spu` → `@TableName("pms_spu")`
- `SpuDetail` → `@TableName("pms_spu_detail")`

**用户模块（6个）：**
- `User` → `@TableName("ums_user")`
- `UserDetail` → `@TableName("ums_user_detail")`
- `UserLoginLog` → `@TableName("ums_user_login_log")`
- `ChangePasswordLog` → `@TableName("ums_change_password_log")`
- `DeliveryAddress` → `@TableName("ums_delivery_address")`
- `RewardPointLog` → `@TableName("ums_reward_point_log")`

**订单模块（3个）：**
- `OmsOrder` → `@TableName("oms_order")`
- `OmsCart` → `@TableName("oms_cart")`
- `OmsOrderItem` → `@TableName("oms_order_item")`

**秒杀模块（3个）：**
- `SeckillSpu` → `@TableName("seckill_spu")`
- `SeckillSku` → `@TableName("seckill_sku")`
- `Success` → `@TableName("success")`

## 四、Spu 实体类字段名与数据库列名不匹配（is_ 前缀）

### 问题描述
修复 `@TableName` 后，SQL 表名正确了，但列名仍然错误。MyBatis-Plus 将 Java 字段 `deleted` 映射为列 `deleted`，但数据库实际列名是 `is_deleted`：

```sql
-- 错误的SQL
SELECT COUNT(*) AS total FROM pms_spu WHERE (category_id = ? AND checked = ? AND deleted = ?)
-- 正确的SQL
SELECT COUNT(*) AS total FROM pms_spu WHERE (category_id = ? AND is_checked = ? AND is_deleted = ?)
```

### 解决方案
在 `Spu.java` 中为 5 个 `is_` 前缀字段添加 `@TableField` 注解：

| Java 字段 | @TableField 值 | 数据库列 |
|---|---|---|
| `deleted` | `@TableField("is_deleted")` | `is_deleted` |
| `published` | `@TableField("is_published")` | `is_published` |
| `newArrival` | `@TableField("is_new_arrival")` | `is_new_arrival` |
| `recommend` | `@TableField("is_recommend")` | `is_recommend` |
| `checked` | `@TableField("is_checked")` | `is_checked` |

## 七、SPU 测试数据 is_checked 默认为 0 导致前端看不到商品

### 问题描述
前端查询条件为 `WHERE is_checked = 1 AND is_deleted = 0`，但 `init-test-data.sql` 的 SPU INSERT 语句未包含 `is_checked` 和 `is_deleted` 字段，默认值均为 0，导致所有商品不显示。

### 解决方案
修改 `database/init-test-data.sql`：
1. 所有 SPU INSERT 语句显式添加 `is_checked=1, is_deleted=0`
2. 将 SPU 数据从 5 条扩充到 20 条，覆盖全部 6 个子分类（手机/平板/笔记本/台式机/男装/女装）
3. SKU 数据从 14 条扩充到 36 条
4. SPU 详情数据从 5 条扩充到 20 条

| `SpuDetail` | Java `detail` ≠ DB `content` |
