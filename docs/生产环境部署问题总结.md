## 二、systemd 服务迁移问题

### 2.1 问题：服务文件路径错误

**现象**：
```
Failed to start mall-ams.service: Unit mall-ams.service not found
```

**原因**：部署脚本中的 glob 路径错误，service 文件放在 `deploy/systemd/` 子目录下，但脚本从 `deploy/` 查找。

**解决**：
```bash
# 手动复制 service 文件到 systemd 目录
for f in /home/ecs-user/deploy/systemd/mall-*.service; do
    cp "$f" /etc/systemd/system/
    chmod 644 "/etc/systemd/system/$(basename $f)"
done
```

### 2.2 问题：JAR 文件名不匹配

**现象**：
```
Unable to access jarfile mall-sso-1.0.0.jar
```

**原因**：不同模块的 JAR 版本号不一致：
- mall-product 和 mall-resource 使用 `1.0.0`
- 其他模块使用 `0.0.1-SNAPSHOT`

**解决**：修改对应 service 文件中的 JAR 路径。

## 三、Seata 配置问题

### 3.1 问题：mall-front 和 mall-search 启动失败

**现象**：
```
org.apache.seata.config.exception.ConfigNotFoundException:
service.vgroupMapping.default_tx_group configuration item is required
```

**原因**：这两个模块的生产配置 `application-prod.yml` 缺少完整的 Seata 配置。

**解决**：在 `application-prod.yml` 中添加完整的 Seata 配置：

```yaml
seata:
  tx-service-group: csmall_group
  service:
    vgroup-mapping:
      csmall_group: default
    grouplist:
      default: ${my.server.addr}:8091
  client:
    undo:
      log-serialization: jackson
```

## 四、Elasticsearch 内存问题

### 4.1 问题：ES 被系统 OOM Killer 杀死

**现象**：
```
elasticsearch.service: Main process exited, code=exited, status=1/FAILURE
elasticsearch.service: Failed with result 'signal'.
```

**原因**：ES JVM 堆内存未配置默认值，系统内存不足时被杀。

**解决**：

1. 创建 ES 堆内存配置文件：
```bash
echo "-Xms512m" > /etc/elasticsearch/jvm.options.d/heap.options
echo "-Xmx512m" >> /etc/elasticsearch/jvm.options.d/heap.options
chmod 644 /etc/elasticsearch/jvm.options.d/heap.options
```

2. 重载 systemd 并启动：
```bash
systemctl daemon-reload
systemctl start elasticsearch
```

## 六、服务器目录结构

### 6.1 /data 目录

| 目录 | 大小 | 所有者 | 用途 |
|------|------|--------|------|
| `/data/jars/` | ~1GB | ecs-user | 微服务 JAR 包、备份、环境变量文件、日志 |
| `/data/nacos/` | ~760MB | root | Nacos 注册/配置中心数据 |
| `/data/seata/` | ~160MB | ecs-user | Seata 分布式事务数据 |
| `/data/elasticsearch/` | ~41MB | elasticsearch | Elasticsearch 数据目录（含 IK 分词器） |
| `/data/frontend/` | ~2.4MB | ecs-user | 前端静态资源 |
| `/data/csmall-upload/` | - | ecs-user | 用户上传文件存储目录 |
| `/data/sessionStore/` | - | root | 会话存储（Redis Session） |

### 6.2 /data/jars 子目录

```
/data/jars/
├── *.jar                    # 微服务 JAR 包（systemd 读取位置）
├── csmall.env               # 共享环境变量（MySQL/Redis/RabbitMQ密码等）
├── start.sh                 # 旧版一键启动脚本（已迁移到 systemd）
├── logs/                    # 应用日志目录
├── backup_*/                # JAR 包备份目录（自动生成）
└── *.sources.jar            # 源码包（可删除节省空间）
```

## 八、注意事项

1. **ES 内存配置**：已限制为 512MB，避免被 OOM Killer 杀死
2. **JAR 上传位置**：统一上传到 `/data/jars/`，不要上传到 `/home/ecs-user/`
3. **systemd 服务**：使用 `systemctl restart` 重启，不要直接 kill 进程
4. **Seata 配置**：新增 Seata 依赖的模块需要同时配置 `application-prod.yml` 和 `application-test.yml`
