### P0.2 幂等性：支付/下单防重提交

**背景**：当前支付接口 `POST /oms/order/pay` 无防重机制，用户快速双击可能重复扣款。

**方案**：Redis + 注解实现通用幂等

**实现步骤**：

1. **定义注解** `@Idempotent`
```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Idempotent {
    String key() default "";      // Redis key 前缀
    long expire() default 5;      // 过期时间（秒）
    String message() default "请勿重复提交";
}
```

2. **实现 AOP 切面** `IdempotentAspect`
```java
@Aspect
@Component
public class IdempotentAspect {
    @Around("@annotation(idempotent)")
    public Object around(ProceedingJoinPoint point, Idempotent idempotent) {
        // 1. 从请求中提取 userId + 方法名 + 参数生成唯一 key
        // 2. Redis SETNX 尝试获取锁
        // 3. 获取成功 → 执行方法 → 返回
        // 4. 获取失败 → 抛出"请勿重复提交"异常
    }
}
```

3. **关键接口加注解**：
   - `POST /oms/order/pay` → `@Idempotent(key = "pay", expire = 10)`
   - `POST /oms/order/add` → `@Idempotent(key = "order", expire = 5)`
   - `POST /seckill/commit` → `@Idempotent(key = "seckill", expire = 3)`

**实施步骤**：

**当前状态**：仅 mall-seckill 模块接入，其他 10 个模块未使用。

**实施**：

| 模块 | 需要限流的接口 | 规则 |

**实现**：

**演示场景**（面试加分）：
- 用 JMeter 压秒杀接口 → Sentinel 限流 → 返回"系统繁忙，请稍后再试"
- Nacos 控制台实时调整限流规则 → 无需重启

### P1.2 普通订单异步处理（MQ）

**当前状态**：只有秒杀用 RabbitMQ，普通下单 `addOrder()` 是同步扣库存 + 同步写库。

**改进**：普通下单也走 MQ 异步化，与秒杀对齐。

```
用户下单 → 同步创建订单(state=0) → 发 MQ 消息 → 返回"下单成功"
  → MQ 消费者 → 扣库存 → 更新订单 state → 完成
```

**改动**：
1. `OmsOrderServiceImpl.addOrder()` 中，扣库存逻辑移到 MQ 消费者
2. 新增 `OrderQueueConsumer` 消费者
3. 下单后订单状态改为"待确认"→ 消费者成功后改为"待支付"

**优点**：削峰填谷 + 解耦 + 提升接口响应速度

### P1.3 日志规范

**当前问题**：
- 18 处 `System.out.println`（P0 已清理）
- 日志级别不统一
- 无 traceId 串联调用链

**实施**：

1. **日志格式统一**（`logback-spring.xml`）：
```xml
<pattern>%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] [%X{traceId}] %-5level %logger{36} - %msg%n</pattern>
```

2. **traceId 生成**（Gateway 层）：
   - 在 Gateway Filter 中生成 `traceId`（UUID 前 8 位）→ 放入请求头 `X-Trace-Id`
   - 每个微服务从请求头读取 → 设置到 MDC (`MDC.put("traceId", traceId)`)
   - 这样一次请求的所有日志都有同一个 traceId

3. **日志级别规范**：
   | 级别 | 场景 |
   |------|------|
   | ERROR | 需要人工介入的异常（支付失败、DB 宕机） |
   | WARN | 业务异常但可降级（库存不足、Token 过期） |
   | INFO | 关键业务流程（下单、支付、发货） |
   | DEBUG | 开发调试信息（SQL、参数） |

4. **敏感信息脱敏**：日志中自动掩码手机号、密码、Token

### P2.2 GitHub Actions CI/CD

**目标**：push 到 master → 自动编译 → 自动部署到测试服务器。

**实施**：
```yaml
# .github/workflows/ci.yml
name: CI
on:
  push:
    branches: [master]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          java-version: 21
          distribution: temurin
      - run: mvn compile -B
      - run: mvn test -B
```

**延伸**（如果有服务器）：
- 编译成功后 SCP JAR 到服务器 → systemd 重启
- 或构建 Docker 镜像 → push 到镜像仓库 → 服务器拉取
