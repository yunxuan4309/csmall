# CoolShark 服务启动指南

> 服务器：阿里云 ECS 8.156.85.160
> 操作系统：Alibaba Cloud Linux 3
> 所有服务部署在 /data 目录下

---

## 一、中间件启动

中间件通过 systemd 或 nohup 管理，以下按依赖顺序启动：

### 1. MySQL

```bash
sudo systemctl start mysqld
# 验证
mysql -u root -p'Csmall2026!Root' -e "SELECT 1;"
```

### 2. Redis

```bash
sudo systemctl start redis
# 验证
redis-cli -a 'Csmall2026!Redis' ping
```

### 3. Nacos

```bash
sudo sh /data/nacos/bin/startup.sh -m standalone
# 验证（启动需要约 10 秒）
sleep 10 && curl -s http://127.0.0.1:8848/nacos/v1/console/health/readiness
```

### 4. Elasticsearch

```bash
sudo systemctl start elasticsearch
# 验证（启动需要约 15 秒）
sleep 15 && curl -s http://127.0.0.1:9200
```

### 5. RabbitMQ

```bash
sudo systemctl start rabbitmq-server
# 验证
rabbitmqctl status | head -5
```

### 6. Seata

```bash
sudo nohup sh /data/seata/bin/seata-server.sh > /data/seata/logs/start.log 2>&1 &
# 验证（启动需要约 10 秒）
sleep 10 && curl -s http://127.0.0.1:7091/health
```

## 二、微服务启动

### 一键启动（推荐）

```bash
bash /data/jars/start.sh
```

### 手动启动

先设置环境变量（每次新开终端都必须执行）：

```bash
export MYSQL_USERNAME=csmall_app
export MYSQL_PASSWORD='Csmall2026!App'
export REDIS_PASSWORD='Csmall2026!Redis'
export RABBITMQ_USERNAME=csmall_app
export RABBITMQ_PASSWORD='Csmall2026!Rabbit'
export ALIYUN_SERVER_IP=127.0.0.1
export RESOURCE_HOST=http://8.156.85.160/
export AI_API_KEY=<你的DeepSeek API Key>
export AI_API_BASE_URL=https://api.deepseek.com
```

按顺序启动各服务：

```bash
# 第一批：Dubbo 提供者（无依赖，先启动）
nohup java -jar /data/jars/mall-ams-webapi-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod --server.port=10003 > /data/jars/logs/mall-ams.log 2>&1 &
nohup java -jar /data/jars/mall-ums-webapi-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod --server.port=10006 > /data/jars/logs/mall-ums.log 2>&1 &
nohup java -jar /data/jars/mall-product-webapi-1.0.0.jar --spring.profiles.active=prod --server.port=9010 > /data/jars/logs/mall-product.log 2>&1 &

# 等待提供者注册到 Nacos
sleep 15

# 第二批：Dubbo 消费者
nohup java -jar /data/jars/mall-sso-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod --server.port=10009 > /data/jars/logs/mall-sso.log 2>&1 &
nohup java -jar /data/jars/mall-order-webapi-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod --server.port=10005 > /data/jars/logs/mall-order.log 2>&1 &
nohup java -jar /data/jars/mall-search-webapi-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod --server.port=10008 > /data/jars/logs/mall-search.log 2>&1 &
nohup java -jar /data/jars/mall-seckill-webapi-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod --server.port=10007 > /data/jars/logs/mall-seckill.log 2>&1 &
nohup java -jar /data/jars/mall-resource-1.0.0.jar --spring.profiles.active=prod --server.port=9060 > /data/jars/logs/mall-resource.log 2>&1 &
nohup java -jar /data/jars/mall-front-webapi-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod --server.port=10004 > /data/jars/logs/mall-front.log 2>&1 &
nohup java -jar /data/jars/mall-ai-webapi-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod --server.port=10010 > /data/jars/logs/mall-ai.log 2>&1 &

# 等待消费者启动
sleep 15

# 第三批：Gateway（最后启动）
nohup java -jar /data/jars/mall-gateway-server-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod --server.port=10087 > /data/jars/logs/mall-gateway.log 2>&1 &
```

## 三、验证所有服务

```bash
for port in 10003 10004 10005 10006 10007 10008 10009 10010 9010 9060 10087; do
  if curl -s -o /dev/null http://127.0.0.1:$port 2>&1; then
    echo "Port $port: RUNNING"
  else
    echo "Port $port: NOT RUNNING"
  fi
done
```

## 四、停止服务

### 停止所有微服务

```bash
pkill -f 'mall-ams-webapi'
pkill -f 'mall-ums-webapi'
pkill -f 'mall-product-webapi'
pkill -f 'mall-sso'
pkill -f 'mall-order-webapi'
pkill -f 'mall-search-webapi'
pkill -f 'mall-seckill-webapi'
pkill -f 'mall-resource'
pkill -f 'mall-front-webapi'
pkill -f 'mall-ai-webapi'
pkill -f 'mall-gateway-server'
```

### 停止中间件

```bash
sudo systemctl stop elasticsearch
sudo systemctl stop rabbitmq-server
sudo sh /data/nacos/bin/shutdown.sh
sudo systemctl stop redis
sudo systemctl stop mysqld
# Seata
pkill -f seata-server
```

## 五、查看日志

```bash

tail -f /data/jars/logs/mall-gateway.log

# 查看最近错误
grep -i error /data/jars/logs/mall-order.log | tail -20

```

## 六、服务器重启后恢复

服务器重启后，需要按以下顺序操作：

1. 启动所有中间件（参见第一节）
2. 启动微服务（参见第二节）
3. 验证所有服务（参见第三节）
