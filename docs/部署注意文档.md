## 二、Sentinel Dashboard

### 2.1 本地启动

**方式一：IDEA Run Configuration（推荐）**

项目 `.idea/runConfigurations/Sentinel_Dashboard.xml` 已配置好，IDEA 右上角下拉菜单选择「Sentinel Dashboard」→ 点绿色三角形启动。

**方式二：命令行**

```bash
java -jar deploy/sentinel-dashboard/sentinel-dashboard-1.8.6.jar
```

**访问**：`http://localhost:8090`
**账号/密码**：`sentinel` / `sentinel`
**JAR 位置**：`deploy/sentinel-dashboard/sentinel-dashboard.jar`（已下载，22MB）
**端口改为 8090 原因**：8080 经常被其他服务占用（Spring Boot 默认端口）

### 2.2 注意事项

- Sentinel Dashboard 是**独立进程**，不随微服务自动启动。没启动 Dashboard 不影响微服务运行，只是看不到可视化控制台。
- Dashboard 默认是**内存存储**，重启后规则会丢失。生产环境需配置 Nacos 持久化（见 2.3）。
- 只有**有流量经过微服务**时，Dashboard 才会显示该服务的数据。刚启动看不到服务是正常的，发几个请求就有了。

### 2.3 规则持久化到 Nacos（生产必须）

当前 Sentinel 规则存在 Dashboard 内存中，Dashboard 重启就没了。生产需改为从 Nacos 读取：

```yaml
# 各模块 application-prod.yml 添加
spring:
  cloud:
    sentinel:
      datasource:
        flow:
          nacos:
            server-addr: ${my.server.addr}:8848
            data-id: ${spring.application.name}-flow-rules
            rule-type: flow
```

这样规则在 Nacos 中持久化，Dashboard 重启不丢失。

### 2.4 Docker 部署

```yaml
# docker-compose.yml
sentinel-dashboard:
  image: bladex/sentinel-dashboard:1.8.6
  container_name: sentinel-dashboard
  ports:
    - "8080:8080"
  environment:
    - JAVA_OPTS=-Dserver.port=8080
    - sentinel.dashboard.auth.username=sentinel
    - sentinel.dashboard.auth.password=sentinel
```

## 三、SkyWalking 链路追踪

### 3.1 当前状态

已下载并可用。启动方式：IDEA Shell Script Run Configuration → `start-skywalking.bat`，或直接双击。
- OAP 后端：端口 11800（gRPC）、12800（HTTP）
- UI 界面：`http://localhost:8088`（端口已在 webapp.yml 中从 8080 改为 8088）
- Agent 探针：`deploy/skywalking/skywalking-agent/skywalking-agent.jar`
- 启动后 UI 显示空白是正常的——需要微服务挂载 Agent 产生流量后才会有数据

### 3.2 Docker 部署（将来）

```yaml
# docker-compose.yml
skywalking-oap:
  image: apache/skywalking-oap-server:9.7.0
  container_name: skywalking-oap
  ports:
    - "11800:11800"   # gRPC（Agent 上报数据）
    - "12800:12800"   # HTTP（UI 查询）

skywalking-ui:
  image: apache/skywalking-ui:9.7.0
  container_name: skywalking-ui
  ports:
    - "8088:8080"     # UI 端口（8088 避免与 Sentinel Dashboard 冲突）
  environment:
    SW_OAP_ADDRESS: http://skywalking-oap:12800
  depends_on:
    - skywalking-oap
```

### 3.3 微服务挂载 Agent

Docker 部署时，每个微服务加 agent 挂载：

```yaml
mall-order:
  image: mall-order:latest
  volumes:
    - ./deploy/skywalking/agent:/skywalking-agent
  environment:
    JAVA_TOOL_OPTIONS: >-
      -javaagent:/skywalking-agent/skywalking-agent.jar
      -DSW_AGENT_NAME=mall-order
      -DSW_AGENT_COLLECTOR_BACKEND_SERVICES=skywalking-oap:11800
```

### 3.4 本地开发（无 Docker）

下载 Java Agent：`https://archive.apache.org/dist/skywalking/java-agent/9.4.0/apache-skywalking-java-agent-9.4.0.tgz`

解压后，每个微服务 VM Options 添加：
```
-javaagent:D:/java/csmall/deploy/skywalking/agent/skywalking-agent.jar
-DSW_AGENT_NAME=mall-order
-DSW_AGENT_COLLECTOR_BACKEND_SERVICES=localhost:11800
```

访问：`http://localhost:8088`

### 3.5 配合 traceId

SkyWalking 的 traceId 和 P1.3 实现的日志 traceId 是同一套体系。SkyWalking Agent 会自动在日志中注入 `[traceId]`，与 Gateway Filter 生成的 traceId 一致。

## 六、启动顺序

```
1. 基础设施：
   MySQL → Redis → Nacos → RabbitMQ → Elasticsearch → Seata → Sentinel Dashboard

2. 独立服务（可并行）：
   mall-resource → mall-ams → mall-ums

3. Dubbo 提供者：
   mall-product（核心，必须在消费者之前启动）

4. Dubbo 消费者（可并行）：
   mall-front → mall-order → mall-search → mall-ai

5. 依赖消费者：
   mall-seckill（等 order/product 就绪）

6. 网关入口：
   mall-sso → mall-gateway-server（最后启动）
```

## 七、常见启动问题

| 问题 | 原因 | 解决 |
|------|------|------|
| RabbitMQ 无法启动 | Windows 计算机名含中文 | `RABBITMQ_NODENAME=rabbit@localhost` |
| Docker Desktop 不可用 | WSL2 未就绪 | 用 Windows 原生中间件，Docker 暂时不用 |
| Maven 编译报"不支持发行版本 21" | IDEA 内置 Maven 用 JDK 17 | Settings → Build → Maven → JDK 改为 21 |
| Redis 不支持 RedisBloom | Windows Redis 3.2.100 | 秒杀用 Redis Set 替代，生产需 RedisBloom |
| Nacos RandomUtils 报错 | 旧版 API 不可用 | 改用 `ThreadLocalRandom` |
| Dubbo 元数据 WARN | 本地无元数据中心 | 不影响功能，忽略 |
| Sentinel Dashboard 看不到服务 | 没流量经过 | 发几个 HTTP 请求就有了 |
| 支付模拟模式 | `alipay.simulated: true` | 测试用，生产改为 `false` |

## 八、支付宝支付配置

### 6.1 模拟模式（当前开发环境）

`application-test.yml` 中 `alipay.simulated: true` → 跳过支付宝 API，直接支付成功。

### 6.2 真实支付（生产切换）

1. `alipay.simulated: false`
2. 确认 `alipay.merchant-private-key-path` 指向正确的私钥文件
3. 网关改为 `https://openapi.alipay.com/gateway.do`（生产网关）
4. `app-id` 改为正式 APPID
5. 密钥配置通过环境变量注入（不要写在 yml 里）

### 6.3 沙箱支付（如果支付宝修复沙箱）

1. `alipay.simulated: false`
2. 确认沙箱密钥文件在 classpath
3. 网关用 `https://openapi-sandbox.dl.alipaydev.com/gateway.do`
4. 沙箱支付页面有已知 JS bug（见 TODO 文件 9.3 节）

## 九、Docker 部署（将来）

### 中间件 docker-compose 模板

```yaml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
    volumes:
      - ./database:/docker-entrypoint-initdb.d
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]

  redis:
    image: redis:7-alpine
    command: redis-server --requirepass ${REDIS_PASSWORD}

  nacos:
    image: nacos/nacos-server:v2.5.2
    depends_on:
      mysql:
        condition: service_healthy
    environment:
      MODE: standalone

  rabbitmq:
    image: rabbitmq:4-management
    environment:
      RABBITMQ_DEFAULT_USER: ${RABBITMQ_USER}
      RABBITMQ_DEFAULT_PASS: ${RABBITMQ_PASS}

  elasticsearch:
    image: elasticsearch:8.6.0
    environment:
      - "ES_JAVA_OPTS=-Xms512m -Xmx512m"
      - xpack.security.enabled=false

  seata:
    image: seataio/seata-server:2.1.0
    depends_on:
      nacos:
        condition: service_started

  sentinel-dashboard:
    image: bladex/sentinel-dashboard:1.8.6

      - "8080:8080"
```

## 十、关键技术决策记录

| 决策 | 原因 |
|------|------|
| Dubbo 3.3.2 替代 Feign | 性能更优，支持多种序列化 |
| MyBatis-Plus IdWorker 替代 Leaf | Leaf 基于 JDK8 + SB2，不再维护 |
| HikariCP 替代 Druid | Druid 曾导致上传请求线程挂死 |
| Seata AT 模式 | `@GlobalTransactional` 保证分布式事务 |
| 图片数据库存相对路径 | 服务层动态拼接，环境切换不用改数据 |
| `@ConditionalOnClass(name = "...")` 替代类字面量 | 避免无对应依赖的模块启动时 ClassNotFoundException |

## 十一、开发注意事项（必读！）

### 11.1 端口规划

| 端口 | 服务 | 冲突处理 |
|------|------|---------|
| 8080 | ~~Sentinel Dashboard~~ → **已改为 8090** | 与 Spring Boot 默认/其他服务冲突 |
| 8088 | SkyWalking UI | 修改 `webapp/application.yml` 中 `serverPort` 从 8080 → 8088 |
| 8848 | Nacos | — |
| 11800 | SkyWalking OAP gRPC | Agent 通过此端口上报数据 |

**每次启动前检查端口占用**：`netstat -ano | grep LISTENING`

### 11.2 SkyWalking 启动顺序（重要！）

**必须先启动 OAP + UI，再启动微服务。** 否则 Agent 连接超时会阻塞启动。

```
1. start-skywalking.bat（等待 OAP 11800 端口就绪）
2. Sentinel Dashboard（IDEA 启动）
3. 微服务（按启动顺序逐个启动）
```

SkyWalking UI 刚启动时数据显示空白是正常的——需要微服务挂载 Agent 并产生流量后才有链路数据。

### 11.3 Git 提交注意事项

| 不要提交 | 原因 |
|---------|------|
| `*.pem` 密钥文件 | 已在 `.gitignore` 排除，已推送到 GitHub 的密钥需重新生成 |
| `deploy/` 目录下二进制文件 | JAR/tar.gz 超过 GitHub 100MB 限制 |
| `*.log` 日志文件 | 运行时产物，已排除 |
| `application-prod.yml` 中的真实密钥 | 生产凭据通过环境变量注入，不要写死在配置里 |

### 11.4 Maven 版本号规范

**内部模块依赖统一用 `${project.version}` 或 `0.0.1-SNAPSHOT`，严禁硬编码 `1.0.0`。**

违反此规则会导致 CI 环境编译失败（找不到对应版本的内部 JAR）。

### 11.5 支付宝支付模式切换

| 环境 | 配置 | 说明 |
|------|------|------|
| 本地开发 | `alipay.simulated: true` | 跳过真实 API，直接支付成功 |
| 沙箱测试 | `alipay.simulated: false` + 沙箱密钥 | 需先将 `alipay_private_key.pem.example` 替换为真实密钥 |
| 正式生产 | `alipay.simulated: false` + 生产密钥 + 生产网关 | 网关改为 `https://openapi.alipay.com/gateway.do` |

### 11.6 幂等性注意事项

`@Idempotent` 注解在 **Redis 不可用时静默失效**（`@ConditionalOnClass` 控制）。如果支付接口的防重逻辑没有生效，首先检查：

1. Redis 是否启动
2. 模块的 `pom.xml` 是否引入了 `spring-boot-starter-data-redis`
3. `application-test.yml` 中 Redis 连接配置是否正确

### 11.7 Sentinel 规则说明

- Sentinel Dashboard 的限流规则是**内存存储**，Dashboard 重启后规则丢失
- 生产环境必须配置 Nacos 持久化（见 2.3 节）
- 没有启动 Dashboard **不影响**微服务的 Sentinel 降级/熔断功能，只是无法可视化管理和查看

### 11.8 Flyway 注意事项

- `spring.flyway.baseline-on-migrate: true` 表示数据库已有表时不报错
- 新增迁移文件命名必须遵循 `V{版本号}__{描述}.sql` 格式（双下划线）
- **迁移文件一旦执行，不可修改内容**——Flyway 会校验 checksum，修改已执行的迁移会导致启动失败
- 测试数据（`init-test-data.sql`、`07-seckill-test-data.sql`）**不走 Flyway**，需手动执行

### 11.9 SQL 注入防护

- 所有自定义 Mapper XML 必须使用 `#{}`（预编译），**严禁 `${}`**（字符串拼接）
- 动态排序/动态列名必须使用 `<choose><when>` 白名单模式，不可直接拼接用户参数
- 新加 Mapper XML 后跑一次全量审计：`grep -rn '\$\{' mall-*/**/src/main/resources/mapper/`
