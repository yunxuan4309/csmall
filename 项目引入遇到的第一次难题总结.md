# Spring Boot 3.x 迁移修复记录

## 概述

本次修复主要解决了项目从 Spring Boot 2.x 迁移到 3.x 过程中遇到的兼容性问题，以及 Maven 多模块项目的依赖配置问题。

---

## 一、Maven 父子模块依赖问题

### 问题描述
- 根 pom.xml 与子模块的 `groupId` 不一致
- 子模块缺少 `<relativePath>` 配置
- 部分依赖版本缺失

### 解决方案
- 统一 groupId 为 `com.cooxiao.mall`
- 为所有子模块添加 `<relativePath>../pom.xml</relativePath>`
- 在根 pom.xml 补充依赖版本：
  - `seata-spring-boot-starter: 1.7.1`
  - `spring-cloud-starter-dubbo: 2021.1`

---

## 二、Spring Boot 3.x 兼容性问题

### 问题描述
Spring Boot 3.x 将 Java EE API 迁移到 Jakarta EE，以下包名发生变化：

| 旧版 | 新版 |
|------|------|
| `javax.servlet.*` | `jakarta.servlet.*` |
| `javax.annotation.PostConstruct` | `jakarta.annotation.PostConstruct` |
| `javax.validation.*` | `jakarta.validation.*` |

### 涉及模块
- mall-ams-webapi
- mall-front-webapi
- mall-order-webapi
- mall-product-webapi
- mall-search-webapi
- mall-seckill-webapi
- mall-sso
- mall-ums-webapi

### 涉及文件类型
- `MyAccessDeniedHandler.java`
- `MyAuthenticationEntryPoint.java`
- `SSOFilter.java`
- `*Controller.java`（使用 HttpServletRequest 的控制器）
- `AutoUpdateTimeInterceptorConfiguration.java`
- `LoginUtils.java`

## 三、Spring Security 6.x 配置问题

### 问题描述
`WebSecurityConfigurerAdapter` 在 Spring Security 6.x 中已被移除。

### 解决方案
重写 Security 配置类，使用 `SecurityFilterChain` Bean：

```java
@Configuration
@EnableWebSecurity
@EnableMethodSecurity(prePostEnabled = true)
public class ResourceWebSecurityConfiguration {

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http.csrf().disable();
        http.cors().configurationSource(corsConfigurationSource());
        http.authorizeHttpRequests()
                .requestMatchers(...).permitAll()
                .anyRequest().authenticated();
        // ...
        return http.build();
    }
}
```

## 四、包路径变更问题

### 问题描述
`mall-common` 模块的类从 `com.cooxiao.mall.common.pojo.domain` 迁移到 `com.cooxiao.mall.common.domain`。

### 解决方案
批量更新所有引用处的 import 语句：
```java
// 旧
import com.cooxiao.mall.common.pojo.domain.CsmallAuthenticationInfo;
// 新
import com.cooxiao.mall.common.domain.CsmallAuthenticationInfo;
```

## 五、库模块打包问题

### 问题描述
库模块继承父 POM 的 `spring-boot-maven-plugin`，执行 repackage 时找不到 main class。

### 解决方案
为以下模块添加跳过 repackage 配置：
- mall-common
- mall-pojo
- mall-ams-service
- mall-front-service
- mall-order-service
- mall-product-service
- mall-search-service
- mall-seckill-service
- mall-ums-service

```xml
<build>
    <plugins>
        <plugin>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-maven-plugin</artifactId>
            <configuration>
                <skip>true</skip>
            </configuration>
        </plugin>
    </plugins>
</build>
```

## 六、版本不一致问题

### 问题描述
webapi 模块依赖 service 模块版本为 `1.0.0`，但 service 模块继承父模块版本 `0.0.1-SNAPSHOT`。

### 解决方案
统一所有 service 模块版本为 `1.0.0`。

## 七、其他问题

### 7.1 删除过时代码
- 删除使用旧版 MyBatis Plus API 的 `CodeGenerator.java` 文件

### 7.2 工具类包迁移
```java
// 旧
import org.apache.commons.lang.StringUtils;
import com.alibaba.nacos.client.naming.utils.RandomUtils;

// 新
import org.apache.commons.lang3.StringUtils;
import org.apache.commons.lang3.RandomUtils;
```

### 7.3 RandomUtils 方法签名变更
```java
// 旧
RandomUtils.nextInt(10000)

// 新
RandomUtils.nextInt(0, 10000)
```

## 八、leaf-server 模块问题

### 问题描述
`maven-dependency-plugin` 的 `copy-dependencies` 在 `compile` 阶段执行失败。

### 解决方案
将执行阶段从 `compile` 改为 `package`：
```xml
<phase>package</phase>
```
