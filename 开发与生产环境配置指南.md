# CoolShark 开发与生产环境配置指南

> 创建时间：2026-05-10
> 用途：说明项目环境配置体系、开发与生产的差异、以及后续开发规范

---

## 一、环境配置体系

项目使用 Spring Boot 的 **Profile 机制** 管理多环境配置：

```
application.yml              ← 所有环境共享的基础配置（端口、JWT、MyBatis、Knife4j 等）
  + application-test.yml     ← 本地开发/测试环境（spring.profiles.active=test）
  + application-prod.yml     ← 生产环境（spring.profiles.active=prod）
```

**核心原则**：基础配置放公共项，环境差异项放对应 profile 文件。profile 文件中的值会**覆盖**基础文件中的同名配置。

### 当前激活方式

各模块 `application.yml` 中默认激活 `test`：

```yaml
spring:
  profiles:
    active: test
```

生产部署时通过启动参数覆盖：

```bash
java -jar app.jar --spring.profiles.active=prod
```

### 中间件地址

| 配置项 | test 环境 | prod 环境 |
|--------|-----------|-----------|
| `my.server.addr` | `localhost` | `${ALIYUN_SERVER_IP:127.0.0.1}` |
| MySQL 地址 | `localhost:3306` | `${my.server.addr}:3306` |
| Redis 地址 | `localhost:6379` | `${my.server.addr}:6379` |
| Nacos 地址 | `localhost:8848` | `${my.server.addr}:8848` |
| Seata 地址 | `localhost:8091` | `${my.server.addr}:8091` |
| ES 地址 | `localhost:9200` | `${my.server.addr}:9200` |
| RabbitMQ 地址 | `localhost:5672` | `${my.server.addr}:5672` |

### 认证信息

| 配置项 | test 环境 | prod 环境 |
|--------|-----------|-----------|
| MySQL 用户名/密码 | `root` / `root` | `${MYSQL_USERNAME}` / `${MYSQL_PASSWORD}` |
| Redis 密码 | 空 | `${REDIS_PASSWORD}` |
| RabbitMQ 用户名/密码 | `guest` / `guest` | `${RABBITMQ_USERNAME}` / `${RABBITMQ_PASSWORD}` |

### 服务行为

| 配置项 | test 环境 | prod 环境 |
|--------|-----------|-----------|
| Gateway 动态路由 | 开启（`locator.enabled: true`） | **关闭**，手动定义路由 |
| Knife4j API 文档 | 可访问（`production: false`） | **屏蔽**（`production: true`） |
| 日志级别 | `trace` / `info` | `info` |
| 文件上传路径 | `E:/csmall-upload`（Windows） | `/data/csmall-upload`（Linux） |
| 资源 URL | `http://localhost:9060/` | `${RESOURCE_HOST}` |

## 三、环境变量说明

生产配置使用 `${ENV_VAR:默认值}` 语法，支持通过环境变量注入敏感信息：

| 环境变量 | 用途 | 默认值 | 使用场景 |
|----------|------|--------|----------|
| `ALIYUN_SERVER_IP` | 服务器地址 | `127.0.0.1` | 单机部署时无需设置，多机部署时设为 VPC 内网 IP |
| `MYSQL_USERNAME` | 数据库用户名 | `root` | 生产环境应使用专用账户 |
| `MYSQL_PASSWORD` | 数据库密码 | `root` | 生产环境必须设置强密码 |
| `REDIS_PASSWORD` | Redis 密码 | 空 | 生产环境必须设置 |
| `RABBITMQ_USERNAME` | RabbitMQ 用户名 | `guest` | 生产环境应使用专用账户 |
| `RABBITMQ_PASSWORD` | RabbitMQ 密码 | `guest` | 生产环境必须设置 |
| `RESOURCE_HOST` | 文件资源访问 URL | `http://127.0.0.1:9060/` | 设为域名，如 `http://coolshark-shop.cn/` |

### 环境变量注入方式

**方式一：启动命令行参数**

```bash
java -jar app.jar \
  --spring.profiles.active=prod \
  --my.server.addr=172.16.0.10 \
  --spring.datasource.password=YourStrongPassword
```

**方式二：系统环境变量**

```bash
export ALIYUN_SERVER_IP=172.16.0.10
export MYSQL_USERNAME=csmall_app
export MYSQL_PASSWORD=YourStrongPassword
export REDIS_PASSWORD=YourRedisPassword
java -jar app.jar --spring.profiles.active=prod
```

**方式三：Nacos 配置中心（推荐后续迁移）**

将敏感配置存入 Nacos，yml 文件中通过 `${xxx}` 引用。这样可以不重启服务动态修改配置。

## 四、Gateway 路由差异

### test 环境（动态路由）

```yaml
spring:
  cloud:
    gateway:
      discovery:
        locator:
          enabled: true    # 所有 Nacos 服务自动可访问
```

开发时方便调试，可以直接通过 `/mall-xxx/` 路径访问任何服务。

### prod 环境（手动路由）

```yaml
spring:
  cloud:
    gateway:
      discovery:
        locator:
          enabled: false   # 关闭动态路由
      routes:
        - id: mall-front
          uri: lb://mall-front
          predicates:
            - Path=/front/**
        - id: mall-order
          uri: lb://mall-order
          predicates:
            - Path=/oms/**
        # ... 只暴露用户需要的服务
```

生产环境只暴露以下服务：
- `mall-front`：前台商品展示
- `mall-order`：订单服务
- `mall-search`：搜索服务
- `mall-sso`：登录认证
- `mall-resource`：文件资源
- `mall-seckill`：秒杀服务

**不对外暴露**的服务（仅通过 Dubbo RPC 内部调用）：
- `mall-ams`：后台管理
- `mall-ums`：用户管理
- `mall-product`：商品管理

## 五、后续开发规范

### 1. 新增配置项的规则

- **所有环境都需要的**：放在 `application.yml` 基础配置中
- **仅开发环境需要的**：只放在 `application-test.yml`
- **仅生产环境需要的**：只放在 `application-prod.yml`
- **两个环境值不同的**：基础配置中不放（或放默认值），分别在 test/prod 中定义

### 2. 敏感信息处理

**绝对不要**将密码、密钥等敏感信息直接写在 yml 文件中并提交到 Git。应该：

1. 使用 `${ENV_VAR:默认值}` 占位符
2. 默认值可以是开发环境的弱密码（如 `root`）
3. 生产环境通过环境变量或启动参数注入真实密码

### 3. 新增模块时

如果新增业务模块，需要同时创建：
- `application.yml` — 基础配置（端口、JWT、MyBatis 等）
- `application-test.yml` — 开发环境（`my.server.addr: localhost`，开发密码）
- `application-prod.yml` — 生产环境（环境变量占位符，`knife4j.production: true`）

### 4. 修改配置时

修改任何配置前先想清楚：
- 这个改动是所有环境都需要，还是只影响某个环境？
- 如果只改了 test 的配置，记得同步检查 prod 是否也需要对应修改

### 5. 本地开发流程

```bash
# 1. 启动本地中间件（MySQL、Redis、Nacos 等）
# 2. 直接运行 Spring Boot 应用，默认激活 test profile
# 3. 访问 http://localhost:{port}/doc.html 查看 API 文档

# 不需要做任何额外配置，test 环境默认就是开发友好的
```

### 6. 部署流程

```bash
# 1. 本地打包
mvn clean package -DskipTests

# 2. 上传 jar 到服务器
scp target/xxx.jar user@server:/opt/csmall/

# 3. 设置环境变量并启动
export ALIYUN_SERVER_IP=127.0.0.1
export MYSQL_USERNAME=csmall_app
export MYSQL_PASSWORD=YourStrongPassword
export REDIS_PASSWORD=YourRedisPassword
export RESOURCE_HOST=http://coolshark-shop.cn/
java -jar xxx.jar --spring.profiles.active=prod
```
